What I work on

Three threads run through the repositories.

The public work clusters into a backend craft, an AI-tooling specialism, and a habit of rebuilding the same product across frameworks to learn each one properly.

// 01

Cloud-native .NET

Microservices on .NET 10 orchestrated with Aspire, meshed with Dapr, and shipped to Azure Container Apps. Database-per-service, a YARP gateway, and CI/CD with quality gates wired in from the start.

// 02

AI developer tooling

Bridging LLMs to real tools through the Model Context Protocol and Microsoft Semantic Kernel — provider-agnostic MCP servers, kernel plugins, and desktop generators that talk to live game-engine editors.

// 03

Full-stack, many frameworks

The same ERP front end rebuilt in Vue, Vite, Angular and Blazor, packaged with Electron, backed by FastAPI and Node, with Flutter on mobile. Breadth chosen on purpose, not by accident.

ERP.Microservices
Flagship

Cloud-native ERP platform

Six independent services — Auth, Billing, Inventory, Orders, Purchasing and Sales — each with its own database, fronted by a YARP gateway and connected through Dapr for service invocation, pub/sub and state. Build, test, coverage and security are enforced through GitHub Actions, SonarCloud and CodeQL, with a generated DocFX documentation site.

Unity-Generator
Flagship

AI asset & code studio for Unity

A local-first desktop app that generates Unity-ready C#, UI prefabs (both uGUI and UI Toolkit), pixel-art sprite sheets, text and audio from your choice of cloud AI providers. An Electron + Vue shell drives a FastAPI backend orchestrated by Semantic Kernel, including a custom Unity MCP server for real-time interaction with the live Editor. Keys stay local; output drops straight into an active Unity project. Full feature parity across Windows, Linux and macOS.

Godot-Generator

AI content generation for Godot

A Blazor WebAssembly app on an ASP.NET Core host (with an optional Electron shell) that connects Semantic Kernel to a Godot MCP server through a published NuGet plugin. Generation is modality-aware — it narrows the exposed MCP tool surface per task across lighting, camera, shaders, nodes and scenes. Built on .NET 10 with xUnit/Moq and an enforced 80% global coverage gate.

image-gen-mcp

Provider-agnostic image MCP server

A Model Context Protocol server that switches between Stability AI and Google Gemini Imagen from runtime tool arguments — the MCP client never notices the change. Hexagonal architecture means a new provider is one interface and zero edits to existing files. Ships with fail-fast config validation, Polly retries, FluentValidation, and distributes as a global dotnet tool that plugs straight into Claude Desktop.

solar-system
Live demo

The Solar System, with the real maths

An interactive 3D Solar System in Three.js: the Sun, eight planets and 54 major moons on true elliptical orbits, solved from J2000.0 orbital elements via Kepler's equation — retrograde moons included. Lock the camera to any body, scrub time from 0× to 100×, and open an info panel with live distance, period and physical data per planet.

The differentiator

An MCP & Semantic Kernel ecosystem.

Rather than one monolith, the AI tooling is built as a constellation of small, composable pieces: Semantic Kernel plugins that speak to providers, MCP servers that expose tools, and a bridge to hold it together. Add a capability by implementing an interface — not by editing what already works.
